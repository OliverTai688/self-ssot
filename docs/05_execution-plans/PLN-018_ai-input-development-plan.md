# AI Input (AI 匯入與思考) 階段開發計畫

本文件定義了 Personal OS 中「AI 匯入與思考」頁面的發展藍圖。核心目標是將傳統的「資料匯入」轉化為「人機協作的脈絡整理」，讓使用者能透過類似 ChatGPT 的介面，專注於資訊的擷取與深層思考。

## 1. 核心願景
> **Capture Cheap, Review Easy.**
> 讓輸入的摩擦力趨近於零，讓整理的智慧感達到最高。AI 不只是存儲，而是理解並提案。

---

## 2. 功能屬性分類

### 2.1 訂閱屬性 (Subscription Attributes)
- **LINE 同步**：持續獲取社交脈絡，識別待辦與人際互動。
- **Google Doc**：匯入長文、文件草稿，提取結構化研究筆記。
- **WebRSS**：主動追蹤外部資訊源，並進行初步摘要與分類建議。

### 2.2 類型屬性 (Type Attributes)
- **多媒體上傳**：圖片 (OCR/摘要)、語音 (逐字稿/重點提取)。
- **內容創作**：Markdown 筆記、快速的想法擷取。
- **AI 紀錄**：回顧先前的 AI 對話，提取共識與決定。

## 3. 階段開發藍圖

### Phase 1: MVP 基礎架構 (已完成)
- **目標**：建立對話式 UI 骨架與模擬匯入機制。
- **核心功能**：
    - [x] 建立類似 ChatGPT 的對話流介面。
    - [x] 整合 `IngestionContext` 以顯示 AI Triage 建議卡片。
    - [x] 實作「模擬匯入」工具列，涵蓋所有核心屬性。
    - [x] 支援手動文字輸入與即時 AI 分析。

### Phase 2: 互動反射與深度審閱 (近期開發)
- **目標**：增強 AI 的互動性與透明度。
- **核心功能**：
    - **混合模式輸入**：支援檔案直接拖放 (Drag & Drop) 到對話框。
    - **澄清式對話 (Reflective Dialog)**：當輸入語意不明時，AI 主動詢問分類建議（例如：「這張照片看起來像收據，要歸入『財務』還是『專案』？」）。
    - **證據標註 (Citations)**：在 AI 建議卡片中，標註原始輸入中觸發判斷的關鍵字。
    - **置信度指示**：對 AI 的判斷給予視覺化信心標籤，低信心項目自動要求人工介入。

### Phase 3: 自動化同步與後台處理 (中期開發)
- **目標**：從「手動點擊」轉向「自動同步」。
- **核心功能**：
    - **真實 API 串接**：實作 LINE Notify 與 Google Doc OAuth 流程。
    - **後台靜默處理**：AI 在背景處理匯入，完成後在對話介面顯示「待審閱摘要」。
    - **批次處理模式**：支援一次確認多筆 AI 建議，減少操作負擔。
    - **RSS 預覽器**：在對話中直接預覽 RSS 摘要，決定是否正式存入系統。

### Phase 4: 脈絡化智慧與長期記憶 (長期目標)
- **目標**：實現跨模組的智慧連結。
- **核心功能**：
    - **跨實體連結 (Contextual Linking)**：AI 能識別輸入內容與現有實體（人、專案、研究）的關聯。
    - **記憶釘選 (Memory Pinning)**：允許使用者將重要的對話或想法釘選，作為長期的 Prompt 參考。
    - **對話重播 (Conversation Replay)**：將「AI 對話紀錄」作為可搜尋的知識庫。
    - **語氣轉換**：根據輸入內容自動調整 AI 的回應語氣（例如：生活事項用溫和語氣，工作事項用專業語氣）。

---

## 4. UI/UX 設計原則

1. **避免「空白恐懼」**：提供啟動引導與按鈕，而非只有一個輸入框。
2. **透明的判斷**：AI 的每一個分類都要解釋「為什麼」。
3. **人類主導 (Human-in-the-loop)**：AI 只提案，決定權永遠在使用者手中。
4. **低認知負荷**：利用動畫、漸進式揭露 (Progressive Disclosure) 來隱藏複雜度。
